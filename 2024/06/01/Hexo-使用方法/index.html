个人网站搭建教程

引言

在数字化时代，个人博客网站已成为展示自我、分享知识、交流想法的重要平台。无论你是专业的开发者、设计师，还是热衷于某一领域的爱好者，拥有一个个人博客网站都能让你更好地发声，与世界建立联系。然而，如果自己从头搭建一个网站，需要良好的网页开发基础和不小的工作量。本文是一篇详细且通俗易懂的教程，即便是没有计算机基础的读者也能轻松搭建一个属于你的个人博客网站。

本教程将涵盖博客网站搭建的各个环节，从配置环境、修改博客，到生成网页、线上部署等，本文将一一为你解析。本文将采用通俗易懂的语言，详细描述实际操作的过程，让你轻松上手，不再为技术难题而烦恼。

Hexo 简介

Hexo 官网：https://hexo.io/zh-cn/index.html

Hexo 是一个博客框架，它可以将易于编辑和阅读的 Markdown 文档转换为网页，并且可以自定义样式主题，它能帮你完成搭建个人网站过程中绝大部分的技术性工作，可以让你不用编写任何代码就完成网站搭建。下面是来自官网的介绍：

Hexo 最初由 Tommy Chen 于 2012 年创建和维护。从那时起，它已经帮助成千上万的人建立了他们梦想中的网站/博客。
Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他标记语言）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

GitHub Pages 简介

要让所有人每时每刻都能看到你的网站，就需要把你的网站放在24小时不间断运行的拥有公网域名的服务器上，但购买/租用服务器和域名都需要不少成本，也比较费事。

GitHub 提供了一项静态站点托管服务——GitHub Pages，你可以免费且轻松地将你的网站部署到 GitHub 的服务器上，所有人都可以随时访问。

修改网页

创建与编辑文章

创建你的第一篇文章

在项目的根目录（即 blog 目录）下打开命令行，输入hexo new “Hello World!”即可创建一篇名为 Hello World! 的文章，新增的文章在目录/source/_posts中，格式为.md。文件的名称通常与文章的标题相同，但由于文件名会成为 URL 的一部分，而 URL 中不能有空格，空格会被替换为-。所以我们刚刚创建的文章的文件名为 Hello-World!.md 。

编辑刚刚创建的文章

用 Markdown 编辑器来编辑刚刚创建的文章，已经了解 Markdown 的可以跳过下面这段。

Markdown 文件需要用专门的编辑器来编辑。常见的 Markdown 编辑器有 Typora, VS Code 等；Markdown 需要使用专门的调整样式的语法，也可以使用 Notion 这样的软件来借助图形界面进行样式调整。关于 Markdown 编辑器以及语法的详细信息请另行搜索。

文章到网页

把文章变成网页

Markdown 格式的文件不能直接作为网页显示，Hexo 的主要功能之一就是把 .md 格式的文件转换为 .html 格式的文件，从而能够作为网页显示。

在 blog 目录下打开命令行，输入并执行指令 hexo clean 或 hexo c （简写），可以让 Hexo 清空之前生成的文件。然后输入并执行指令 hexo generate 或 hexo g （简写），可以让 Hexo 根据已有的所有 Markdown 文件生成相应的 HTML 文件。

运行网页进程

在 blog 目录下打开命令行，输入并执行指令 hexo server 或 hexo s （简写），可以让 Hexo 将网页服务在本地的 4000 端口上运行起来。启动后在浏览器地址栏输入 localhost:4000 并按下回车，就能看到网站页面，其中包含了前面部分新添加的文章。若要结束进程，在刚刚的命令行窗口按 Ctrl + C 即可结束进程。

但此时你的网页只有你自己在启动本地进程后能够看到，别人还访问不了你的网站。若要让他人访问，需要把网页部署到公网上，见 部署网页 部分。

GitHub 项目设置

创建项目仓库

进入 GitHub 并登录，点击 右上角自己的头像 → Your Repositories ，再在跳转到的页面中点击绿色的 New 按钮，进入创建仓库页面。在 “Repository name” 这一项中填入 用户名.github.io （格式必须按照这样），其中“用户名”替换为你自己 GitHub 账号的用户名。Description 可填可不填，Description 下面的选项一定要选择 Public。下面的创建 README 也是可选可不选。完成后点击最下面的 Create repository 按钮来创建仓库。

设置 GitHub Pages

点击 右上角自己的头像 → Your Repositories，再点击刚刚创建的项目名称，然后在页面上方的导航栏中选择 Settings。跳转到项目设置页面后，在左侧边栏中点击 Pages，在标题 Build and deployment 下面的 Source 选择 Deploy from a branch, 在 Branch 选择 master 然后点击 save.
